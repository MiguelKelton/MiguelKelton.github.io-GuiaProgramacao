GeneXus

O que é

GeneXus é uma plataforma de desenvolvimento de software de baixo código (low-code) impulsionada por inteligência artificial que automatiza e simplifica a criação, evolução e manutenção de aplicações empresariais. Essencialmente, é um sistema que permite que desenvolvedores criem software de forma mais rápida e eficiente, utilizando representação de conhecimento e gerando código nativo para diversas plataformas, como web, dispositivos móveis e sistemas Windows.

Knowledge Base (KB)

Base de conhecimento
A KB é o projeto principal em GeneXus. É onde você desenvolve toda a sua aplicação. Em resumo, a KB é o coração do seu sistema dentro do GeneXus.

Objetos em genexus

Transaction {

O que é um Transaction?

Um Transaction representa uma entidade do mundo real, como por exemplo: Cliente, Produto ,Pedido, Funcionário
Quando você cria uma transação, você está dizendo ao GeneXus:
“Quero armazenar informações sobre X, com tais atributos, e com tais relações.”

O que ela faz?

Ao criar uma Transaction, GeneXus: Gera automaticamente a estrutura da tabela no banco de dados. Cria a interface de cadastro (tela) para inserir, editar e excluir dados. Define regras como chave primária, relacionamento e integridade referencial.
Não precisa escrever SQL nem montar telas manualmente. Você modela no Transaction e o GeneXus: Cria a tabela no banco, Cria os relacionamentos corretos, Gera a interface de cadastro automaticamente Aplica as regras de integridade

}

Structure {

O que é a aba Structure?

A aba Structure (estrutura) em uma Transaction do GeneXus serve para definir os atributos e a hierarquia de dados que formarão a transação — ou seja, a estrutura que o GeneXus vai transformar em tabelas no banco.

Atributos (Attributes)

Para cada Objeto transação, você tem que definir o atributos ou campos que descrevem aquele objeto da realidade.
São os campos da transação (como colunas em uma tabela). Exemplo: Id, Nome, Email, CPF.

Primary Key

A chave primária é o campo (ou conjunto de campos) que identifica unicamente cada registro de uma tabela (ou "Transaction" no GeneXus).

Ela é atribuida automaticamente ao primeiro campo criado de uma Transaction

Também é possível clicar com o botão direito em um atributo e marcar como chave primária (Key), caso deseje alterar a definição.

Data Types (Tipo de dados) {

em GeneXus, os Data Types (tipos de dados) definem que tipo de informação um atributo ou variável pode armazenar. Eles são essenciais para controlar validação, formato, armazenamento no banco de dados e apresentação na interface.

Categorias de Data Types:

em GeneXus, os Data Types (tipos de dados) definem que tipo de informação um atributo ou variável pode armazenar. Eles são essenciais para controlar validação, formato, armazenamento no banco de dados e apresentação na interface.

Numeric:

Representa números com ou sem casas decimais, usado para valores como preços, quantidades, porcentagens, etc.

Exemplo: Numeric(5,2)

Varchar:

É um tipo de dado de texto de tamanho variável, ideal para nomes, descrições curtas e dados onde o tamanho pode variar bastante.

Exemplo: VarChar(n)

Character:

Representa uma cadeia de texto de tamanho fixo. Ou seja, sempre reserva n posições, mesmo que o texto seja menor.

Exemplo: Character(10)

Date:

Armazena somente a data, sem hora. Muito usado para datas de nascimento, datas de vencimento, datas de cadastro, etc.

Exemplo: dd/mm/yyyy

Boolean:

Tipo lógico, que armazena apenas dois valores:

Exemplo: True / False

}

Foreign Key

A chave estrangeira é um atributo que referencia a chave primária de outra transação. Ela serve para relacionar duas tabelas.

Dominio

O que é

Domínios em GeneXus são definições reutilizáveis de tipos de dados
Tipo de dado (por exemplo: Character, Numeric, Date, etc.)
Tamanho e formato
Regras de validação (opcional)
Máscara de exibição, descrição, etc.

Assim, em vez de configurar esses detalhes repetidamente para cada atributo, você os define uma vez no domínio e reutiliza onde quiser.

Como definir um dominio

Id=Numeric(4,0)
